# Глава 141: PINN и Блэк-Шоулз -- Простое объяснение

## Представь, что ты покупаешь билеты на концерт...

Давай разберемся с физически обоснованными нейросетями для оценки опционов через простую историю.

---

## Задача о билете на концерт

### Что такое опционы?

Представь: твоя любимая группа объявила концерт через 6 месяцев. Билеты будут стоить 10 000 рублей. Но ты пока не уверен, хочешь ли идти.

Друг предлагает тебе сделку:

```
"Заплати мне 1 000 рублей сейчас, и я гарантирую тебе
ПРАВО купить билет за 10 000 в любой момент в ближайшие 6 месяцев."
```

Это **опцион**! Ты платишь небольшую сумму сейчас за *право* (но не обязанность) купить что-то позже по фиксированной цене.

```
Если цена билета ВЫРАСТЕТ до 15 000:
  - Ты используешь опцион: покупаешь за 10 000, экономишь 5 000
  - Твоя прибыль: 5 000 - 1 000 (стоимость опциона) = 4 000

Если цена УПАДЕТ до 6 000:
  - Ты НЕ используешь опцион (зачем платить 10 000, если стоит 6 000?)
  - Твой убыток: 1 000 (то, что заплатил за опцион)

Если цена останется 10 000:
  - Ты НЕ используешь опцион (нет выгоды)
  - Твой убыток: 1 000
```

## Главный вопрос: какова справедливая цена?

Вот загадка: **Сколько должен стоить этот опцион?**

Если слишком дешево (100 рублей) -- твой друг потеряет деньги. Если слишком дорого (5 000 рублей) -- никто не купит.

Два знаменитых математика, **Фишер Блэк** и **Майрон Шоулз**, вывели формулу для этого в 1973 году. За это дали Нобелевскую премию!

### Формула Блэка-Шоулза (упрощенно)

Их открытие: цена опциона зависит от:

```
1. Текущая цена билета:         10 000 руб.
2. Гарантированная цена покупки: 10 000 руб. ("страйк")
3. Время до концерта:           6 месяцев
4. Насколько дико скачут цены:  "Волатильность" (напр., 20%)
5. Процентная ставка:           Если положить деньги в банк
```

Они создали математическое уравнение (дифференциальное уравнение в частных производных), которое говорит:

```
"Цена опциона должна меняться во времени так, чтобы
идеально балансировать риск -- бесплатный сыр только в мышеловке!"
```

## Появляется: Нейросеть, которая знает физику

### Что такое PINN?

**Физически обоснованная нейросеть** (PINN) -- это как ученик, который должен сдать ДВА экзамена:

```
Экзамен 1 (по данным):
  "Совпадает ли ответ с известными фактами?"
  Пример: В день концерта опцион на билет за 15 000
  при страйке 10 000 стоит ровно 5 000.

Экзамен 2 (по физике/математике):
  "Подчиняется ли ответ правилам?"
  Пример: Цена опциона должна удовлетворять уравнению
  Блэка-Шоулза В КАЖДОЙ точке по времени и цене.
```

### Обычные нейросети vs PINN

```
ОБЫЧНАЯ нейросеть (учится только по данным):
+-----------------------------------------------------+
| Дайте мне тысячи примеров правильных цен,           |
| и я выучу закономерность.                            |
|                                                       |
| Проблема: нужно МНОГО данных. Может выучить          |
| неправильные паттерны. Может грубо ошибаться         |
| там, где нет обучающих данных.                        |
+-----------------------------------------------------+

PINN (учится по данным И правилам):
+-----------------------------------------------------+
| Дайте мне математические правила (уравнение          |
| Блэка-Шоулза), несколько граничных фактов,           |
| и я вычислю правильную цену ВЕЗДЕ.                   |
|                                                       |
| Преимущество: нужно мало данных. Всегда подчиняется  |
| правилам. Умная экстраполяция.                       |
+-----------------------------------------------------+
```

## Аналогия с кулинарией

### Как учится PINN?

Представь, что ты учишься печь торт:

**Шаг 1: Знать рецепт (уравнение)**
```
Рецепт гласит: "Каждый торт должен подниматься равномерно"
На языке математики: dV/dt + 0.5*sigma^2*S^2*d^2V/dS^2 + r*S*dV/dS - r*V = 0

(Не переживай из-за формул -- это просто "рецепт")
```

**Шаг 2: Знать края (граничные условия)**
```
"Если нет ингредиентов (S=0), торта не будет (V=0)"
"Если ингредиентов бесконечно, торт очевиден"
```

**Шаг 3: Знать финал (терминальное условие)**
```
"В конце (день концерта) торт готов:
 он стоит max(цена билета - 10 000, 0)"
```

**Теперь PINN тренируется так:**

```
Попытка 1: Испечь торт (угадать цену)
  Шеф проверяет: "Следует ли рецепту?"     -> Нет! Штраф: 50
  Шеф проверяет: "Края правильные?"        -> Нет! Штраф: 30
  Шеф проверяет: "Финал правильный?"       -> Нет! Штраф: 40
  Итого: 120 (плохо!)

Попытка 1000: Новый торт
  Шеф проверяет: "Следует ли рецепту?"     -> Почти! Штраф: 2
  Шеф проверяет: "Края правильные?"        -> Да! Штраф: 0.1
  Шеф проверяет: "Финал правильный?"       -> Да! Штраф: 0.5
  Итого: 2.6 (отлично!)

Попытка 5000: Почти идеальный торт!
  Итого: 0.01 (нейросеть научилась!)
```

## А что с криптовалютами?

### Опционы на Bitcoin на Bybit

Та же идея работает для Биткоина! Bybit (криптобиржа) предлагает опционы на BTC, ETH и SOL.

В чем разница?

```
Опционы на акции:
  - Колебания цен: ~20% в год
  - Торговые часы: с 9:30 до 16:00, будние дни
  - Существуют с 1973 года

Крипто-опционы:
  - Колебания цен: ~80% в год (БЕЗУМИЕ!)
  - Торговые часы: 24/7, без выходных
  - Относительно новые
```

PINN работает одинаково для обоих -- просто подставляем другие числа для волатильности!

## Греки: Суперспособности опционов

Когда ты обучаешь PINN, ты получаешь бонусную информацию бесплатно! Это называется "греки":

```
Дельта: "Если акция вырастет на 1 рубль, насколько изменится опцион?"
  Как: "Если дождь усилится на 1 мм, насколько быстрее наполнится ведро?"

Гамма: "Как быстро меняется Дельта?"
  Как: "Ведро наполняется все быстрее и быстрее?"

Тета: "Сколько стоимости опцион теряет каждый день?"
  Как: "Мороженое тает... как быстро?"

Вега: "Если цены станут более волатильными, что случится с опционом?"
  Как: "Если погода станет более непредсказуемой,
       нужен ли мне зонт побольше?"
```

PINN вычисляет это автоматически благодаря тому, как работают нейросети (так называемое "автоматическое дифференцирование" -- тот же математический прием, который позволяет нейросетям учиться).

---

## Почему это круто?

```
Традиционный метод (конечные разности):
  Как рисование на бумаге в клетку -- работает, но медленно
  и ограничено конкретной сеткой

Метод Монте-Карло:
  Как подбрасывание миллиона монет -- работает, но шумно

Формула Блэка-Шоулза:
  Как волшебная формула -- работает, но только для простых случаев

PINN:
  Как умный ученик, знающий правила
  - Работает для ЛЮБОГО случая (даже экзотических опционов)
  - Греки достаются бесплатно
  - Быстрый после обучения
  - Может работать с несколькими акциями одновременно
```

## Попробуй сам!

### Самый простой пример

```python
# Создаем "ученика" (нейросеть)
мозг = НейроСеть(входы=2, скрытые=[128,128,128,128], выход=1)

# "Правила", которым он должен следовать:
# Правило 1: Уравнение Блэка-Шоулза (рецепт)
# Правило 2: При истечении call = max(S - K, 0) (финал)
# Правило 3: При S=0 call = 0 (край)

# Тренируемся 10 000 раундов
for раунд in range(10000):
    # Проверка: следует ли рецепту?
    ошибка_рецепта = проверить_блэк_шоулз(мозг)

    # Проверка: края правильные?
    ошибка_краев = проверить_границы(мозг)

    # Проверка: финал правильный?
    ошибка_финала = проверить_выплату(мозг)

    # Итоговая оценка (чем меньше, тем лучше)
    итого = ошибка_рецепта + ошибка_краев + ошибка_финала

    # Учимся на ошибках
    мозг.улучшить(итого)

# Теперь мозг знает цены опционов!
цена = мозг.предсказать(цена_акции=100, время=0)
# -> примерно $10.45 (совпадает с формулой!)
```

## Итоги

```
Что мы узнали:
1. Опционы = право купить/продать по фиксированной цене
2. Блэк-Шоулз = математические правила справедливой цены опционов
3. PINN = нейросеть, которая учится, следуя правилам
4. Греки = бонусная информация о том, как меняются цены опционов
5. Работает и для акций, И для криптовалют!

Ключевая идея:
  Вместо того чтобы решать сложное математическое
  уравнение напрямую, мы обучаем нейросеть, которая
  ОБЯЗАНА подчиняться уравнению. Это как учить ученика
  законам физики вместо того, чтобы давать ему
  шпаргалку с ответами.
```
